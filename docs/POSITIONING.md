# Positioning

## One-Line Positioning

Spider Framework Suite is a multi-runtime crawler platform with one operator model across Java, Go, Python, and Rust.

## What It Is

- not a single-language crawler library
- not only a browser automation wrapper
- not only a pile of internal demos

It is:

- one shared contract
- four native runtime implementations
- one evidence-driven release and verification lane

## Why It Matters

Most crawler frameworks optimize one language.
This suite optimizes one platform model across four languages.

That means the public story should stay simple:

- same task model
- same config model
- same control-plane model
- comparable evidence and artifacts

## Best-Fit Narrative

- Python for rapid extraction and AI-assisted research
- Go for lightweight concurrent services
- Rust for typed high-reliability runtime cores
- Java for workflow-heavy enterprise integrations

The suite-level value is the shared contract across all four.
