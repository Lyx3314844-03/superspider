# Scrapy-Style Authoring

本仓库现在把四个运行时都补出了一层统一的、接近 Scrapy 的“写爬虫”接口。

目标不是逐字复制 Scrapy 内核，而是统一这些高频作者心智模型：

- `Spider`
- `Request`
- `Response`
- `Item`
- `Selector`
- `FeedExporter`
- `CrawlerProcess`

模块：

- `pyspider.spider.spider`
- 顶层包也可直接从 `pyspider` 导入

## 统一示例形态

所有运行时都鼓励用同一类结构：

1. 定义 `Spider`
2. 配置 `start_urls`
3. 在 `parse` 中产出 `Item`
4. 用 `response.follow(...)` 跟进下一页
5. 用 `FeedExporter` 导出结果

## 当前边界

这层是 authoring API，不是要把四套 runtime 强行重写成同一个实现。

保留现状：

- 浏览器能力仍走各运行时原生 browser 模块
- 分布式能力仍走各运行时原生 distributed 模块
- anti-bot 仍走各运行时原生 anti-bot 模块

推荐做法：

- 日常站点采集先用这层统一 API
- 遇到浏览器、反爬、分布式需求，再下沉到对应 runtime 的原生能力
- Python project runner 现在也支持在统一 API 内做混合路由：`Request(..., meta={"runner": "browser"})`
- 请求级 browser 参数支持：`meta["browser"] = {"session": "...", "wait_until": "...", "wait_for_selector": "...", "timeout_seconds": 15}`（也兼容 `browser_*` 平铺键）

## Python Plugin SDK

Python 现在已经有 project 级插件参考实现：

- 基类：`pyspider.spider.spider.ScrapyPlugin`
- 默认发现：`plugins.py`
- 显式注册：`scrapy.plugins: ["plugins:ProjectPlugin"]`
- 可选 manifest：`scrapy-plugins.json`

当前支持的钩子：

- `configure(config)`
- `provide_pipelines()`
- `provide_spider_middlewares()`
- `provide_downloader_middlewares()`
- `on_spider_opened(spider)`
- `on_spider_closed(spider)`
- `process_item(item, spider)`

当前状态：

- Python 已经是 reference implementation
- Go 已经有 project 级 plugin 注册链路
- Rust 已经有 project 级 plugin 注册链路
- Java 已经有 project 级 plugin 注册链路
- Go / Rust / Java 当前都支持 plugin 生命周期钩子、pipeline 注入、item 处理钩子
- Python 仍然是最完整的 project 级 reference implementation

## Go / Rust / Java Plugin SDK

当前三门语言都已经支持最小 plugin 接口，并接入了 `CrawlerProcess`。

支持能力：

- plugin `configure(config)`
- spider 启动前准备
- spider 打开/关闭生命周期
- plugin 注入 pipeline
- plugin 注入 spider middleware
- plugin 注入 downloader middleware
- plugin 处理 item
- `Request.meta["runner"]` 风格的 browser/http 选择已经补到 shared scrapy core

当前边界：

- Go:
  - 已有 project 级注册链路，project runner 会加载已注册插件
  - 支持 `scrapy-plugins.json`
- Rust:
  - 已有 project 级注册链路，project runner 会加载已注册插件
  - 支持 `scrapy-plugins.json`
- Java:
  - 已有 project 级注册链路，project runner 会加载已注册插件
  - 支持 `scrapy-plugins.json`

## Starter Kits

现有 starter 已经补了 scrapy 风格源码入口：

- `examples/starters/pyspider-starter/scrapy_demo.py`
- `examples/starters/gospider-starter/main.go`

对应运行脚本：

- `run-scrapy.sh`
- `run-scrapy.ps1`

AI starter files generated by `scrapy init`:

- `ai-schema.json`
- `ai-job.json`
- `ai-blueprint.json`
- `ai-extract-prompt.txt`
- `ai-auth.json`

These provide a shared starting point for running the unified `ai` command or an `ai` runtime job after project scaffolding.

`ai-auth.json` supports:

- `headers`
- `cookies`
- `storage_state_file`
- `cookies_file`
- `session`
- `actions`
- `action_examples`
- `node_reverse_base_url`
- `capture_reverse_profile`

Supported `actions` currently include:

- `if`
- `goto`
- `wait`
- `click`
- `type`
- `otp`
- `mfa_totp`
- `submit`
- `wait_network_idle`
- `captcha_solve`
- `captcha_wait`
- `select`
- `hover`
- `scroll`
- `eval`
- `assert`
- `save_as`
- `listen_network`
- `reverse_profile`
- `reverse_analyze_crypto`
- `screenshot`

Example:

```json
{
  "actions": [
    { "type": "goto", "url": "https://example.com/login" },
    { "type": "type", "selector": "#username", "value": "demo" },
    { "type": "type", "selector": "#password", "value": "secret" },
    {
      "type": "if",
      "when": { "selector_exists": "#otp" },
      "then": [
        { "type": "mfa_totp", "selector": "#otp", "totp_env": "SPIDER_AUTH_TOTP_SECRET" }
      ]
    },
    {
      "type": "if",
      "when": { "selector_exists": ".cf-turnstile,[data-sitekey]" },
      "then": [
        {
          "type": "captcha_solve",
          "challenge": "turnstile",
          "selector": ".cf-turnstile,[data-sitekey]",
          "provider": "anticaptcha",
          "save_as": "captcha_token"
        }
      ]
    },
    { "type": "submit", "selector": "#password" },
    { "type": "wait_network_idle" },
    { "type": "reverse_profile", "save_as": "reverse_runtime" },
    { "type": "assert", "url_contains": "/dashboard" },
    { "type": "save_as", "value": "url", "save_as": "final_url" }
  ]
}
```

Project runner policy:

- Python 继续使用原生 project runner
- Go / Rust / Java 现在优先执行 `scrapy-project.json` 中声明的 project runner artifact
- 如果 artifact 不存在，则回退到 built-in metadata runner
- Go / Rust / Java 不再在 `scrapy run --project` 时动态编译项目源码
