# 媒体与平台支持

## 共享媒体能力

四个框架都可以在 GitHub 文档中公开描述为覆盖以下媒体问题域：

- HLS / `m3u8`
- DASH / `mpd`
- FFmpeg 协同处理
- DRM 检测
- 视频平台解析
- 媒体下载流程
- 清单解析与合并

## 共享平台支持

当前文档面可统一描述的主要平台包括：

- YouTube
- Bilibili
- IQIYI
- Tencent Video
- Youku
- Douyin

## 适合 GitHub 公开的表达方式

建议不要把四个框架宣传成“只会网页抓取”的传统爬虫。

更准确的定位是：

- 通用网页抓取
- 动态页面抓取
- 加密站点处理
- 视频站点解析
- 媒体资源下载与处理

## 四个框架在媒体面的共同卖点

- 不只抓 HTML
- 也能处理视频站点和动态媒体页
- 能把浏览器、网络、媒体清单解析结合起来
- 能接 FFmpeg 形成完整媒体处理链

## 框架差异

### PySpider

- 更适合做项目制媒体抓取工作流
- 更适合和 Python 工具链联动

### GoSpider

- 更适合高并发和服务端媒体任务
- 更适合二进制部署的下载器场景

### RustSpider

- 更适合高性能、严格发布边界的媒体运行时

### JavaSpider

- 更适合已有 Java 环境中的媒体解析与自动化流程

## GitHub 建议定位

如果这个 GitHub 仓库只保留 Markdown，媒体能力建议作为“第二核心卖点”展示：

1. 四语言统一爬虫能力
2. Node 逆向与反爬能力
3. 媒体站点解析与下载能力
