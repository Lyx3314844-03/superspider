# JavaSpider 能力说明

## 框架定位

`JavaSpider` 是四个框架里最偏 Java 生态、Maven/JAR 交付、浏览器工作流和企业集成的一档。

它适合：

- Java 团队
- Maven/JAR 工程
- 浏览器流程密集型任务
- 审计、集成、企业内部系统对接

## 核心能力

- Maven / JAR 打包
- 浏览器工作流
- 反爬能力
- Node.js 逆向接入
- 媒体解析
- 项目运行与 CLI
- 审计与控制台能力

## 浏览器与动态页面

`JavaSpider` 的浏览器能力更适合公开描述为：

- Selenium 主线浏览器自动化
- Playwright 辅助兼容能力
- 浏览器表单与工作流驱动
- 动态页面处理
- 自动化流程执行

它适合：

- Java 生态已有浏览器自动化栈
- 企业内部流程自动化
- 强调 Maven 构建和 JAR 交付

## Node.js 逆向能力

`JavaSpider` 的 `NodeReverseClient` 接口面很完整，而且有较强的本地静态分析兜底。

支持：

- crypto analyze
- encrypt / decrypt
- AST extract / analyze
- function call
- JS execute
- browser simulate
- Webpack analyze
- anti-bot detect / profile
- fingerprint spoof
- TLS fingerprint
- canvas fingerprint
- signature reverse

本地分析同样覆盖：

- 算法识别
- mode 识别
- dynamic key sources
- runtime algorithm selection
- obfuscation loaders
- crypto sinks
- key flow candidates
- key flow chains
- riskScore / reverseComplexity
- recommendedApproach

## 加密站点处理链

`JavaSpider` 已经有比较明确的“加密站点处理器”公开描述面。

适合在 GitHub 上展示为：

- 页面反爬画像
- 浏览器环境模拟
- 加密检测
- 混淆代码执行
- 解密数据提取
- Webpack 分析
- 签名逆向

## 工程与企业侧优势

- Maven / JAR 打包清晰
- 适合接入企业 Java 系统
- 审计与浏览器工作流更容易被 Java 团队接受
- 更适合做内部平台和企业自动化组件

## 最适合的 GitHub 公开定位

- 面向 Java / Maven / JAR 交付的浏览器与反爬爬虫框架
- 兼顾 Node 逆向、加密站点处理和企业系统集成
- 适合需要 Java 工程规范和企业交付形态的团队
